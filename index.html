Installation and Set Up

Ensure you have python3 installed: sudo apt install python3
Ensure you have pip installed: sudo apt install python3-pip

Due to the Guelph linux servers being externally managed, a virtual enviornment on the sever is created and worked in, in order to install and use pytest. To install a virtual environment type the following into the correct directory: python3 -m venv myenv

In order to activate the virtual environment, in order to allow you to work within a controlled environment where installed packages won't affect the system-wide Python installation, type: source myenv/bin/activate

In order to install the Pytest framework inside the activated virtual environment, type: pip install pytest

Once you are done developing or testing within the virtual environment, its good practice to deactivate the environment. Although this is optional, its highly recommended. To do so, type: deactivate

Module Overview: Writing Pytest for the win_check Function

In this module, we'll delve into the essentials of writing effective Pytest cases for the win_check function in a Tic Tac Toe game. The win_check function determines whether a player has won the game based on the current state of the game board.

Key Concepts Covered:

  • Parameterized Testing:
    Learn how to use @pytest.mark.parametrize to run multiple test cases with different inputs on the win_check function, ensuring comprehensive test coverage.
  • Test Case Structure:
    Understand the structure of a Pytest test case, including the use of the pytest.mark.parametrize decorator, which simplifies testing for various scenarios.
  • Assertion in Pytest:
    Explore the assert keyword within Pytest, a fundamental tool for verifying that the win_check function returns the expected results for each test case.
  • Expected Failures:
    Discover how to intentionally create a failing test case (as seen in Test Case 5), providing insights into the importance of accurate expected results.
  • Interpreting Test Results:
    Learn to interpret Pytest output, differentiating between passed and failed tests, and identifying the specific test cases that require attention.

By the end of this module, you'll have a solid understanding of how to structure and execute Pytest cases, ensuring the reliability and accuracy of the win_check function in your Tic Tac Toe game.
