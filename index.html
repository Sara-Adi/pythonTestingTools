Module Overview: Understanding the Code: tic_tac_toe_ai.py

1. Default Function:

- Prints a welcome message to initiate the Tic Tac Toe game.

2. Rules Function:

- Displays the layout of the Tic Tac Toe board with positions numbered from 1 to 9.

- Informs players that they need to input a number (1-9) to make a move.

3. Display Board Function:

- Takes the current state of the Tic Tac Toe board and prints it, showing the positions and markers (X, O).

4. Computer AI Function (CompAI):

- Implements a basic AI for the computer player to make strategic moves.

- Prioritizes winning moves, blocking opponent wins, choosing open corners, and selecting available edges.

- Utilizes random selection for certain scenarios.

5. Select Random Function:

- Generates a random selection from a given list.

6. Place Marker Function:

- Places the marker (X or O) on the board at the specified position.

- Checks if the position is valid (between 1 and 9).

7. Win Check Function:

- Checks if the current player has achieved a win based on various possible patterns on the board (horizontal, vertical, diagonal).

8. Full Board Check Function:

- Determines if the Tic Tac Toe board is full, indicating a draw.

9. Replay Function:

- Asks users if they want to play the game again.

- Accepts input (Y/N) and returns True if the answer starts with 'Y'.

Main Program:

  • Initiates the game loop, allowing continuous gameplay until the users decide to exit.
  • Sets up the game with initializations such as player names, marker choices (X and O), and starts the turn-by-turn gameplay.
  • Displays the game outcome, including the winner or a draw, and prompts users to replay.

Note:

  • The game is currently set up to be played between two computer players (Computer1 and Computer2).
  • The logic allows for both players to make strategic moves, and the game continues until the players decide not to replay.

Basics of Pytest

Introduction to Pytest:
What is Pytest?

A Python testing framework designed for straightforward unit tests, featuring fixtures and parameterized testing capabilities.

Why should we use Pytest?

Simply because it is easy to use, helps manage test dependencies with fixtures, and offers built-in assertions for writing clear and effective test code. Not to mention, Pytest has one of the most active community support that makes it widely used in the Python world.

Assertions and Test Outcomes:
What are assertions?

assert in Python are statements used to check if a given condition is true, commonly used in testing to verify the conditions of a program. If the assertion fails, it will raise AssertionError providing a clear indication of the issue. For example, 

                        
                            def test_addition():
                            result = 2 + 2
                            assert result == 4, "Addition failed"

This test checks for the result of a simple addition operation, ensuring that it equals 4 and raising an assertion error with the message "Addition failed" if the condition is not met.

Overview of different assertions in Pytest

There is a variety of assertions that you can use to check conditions and verify expectations in your tests. Here's an overview of some common assertions in Pytest:

Equality and Inequality:

                        
                            assert a == b    # Asserts that a is equal to b
                            assert a != b    # Asserts that a is not equal to b

Identity and Non-Identity:

                        
                            assert a is b    # Asserts that a is the same object as b
                            assert a is not b # Asserts that a is not the same object as b

Containment:

                        
                            assert item in sequence    # Asserts that item is in the sequence
                            assert item not in sequence # Asserts that item is not in the sequence

Greater and Less Than:

                        
                            assert a > b   # Asserts that a is greater than b
                            assert a < b   # Asserts that a is less than b

Greater/Less Than or Equal:

                        
                            assert a >= b  # Asserts that a is greater than or equal to b
                            assert a <= b  # Asserts that a is less than or equal to b
Understanding test outcomes (pass, fail, skip)

In Pytest, test outcomes are categorized into three main statuses: pass, fail, and skip. These outcomes provide information about the results of the test cases:

  • Pass:

    • A test passes when all assertions within the test case are true, and there are no unhandled exceptions.

  • Fail:

    • A test fails when at least one assertion within the test case evaluates to false or if an unexpected exception occurs.

  • Skip:

    • A test is skipped when it is excluded from execution, usually due to some condition not being met (for instance, the environment is not suitable for the test).

Example of showcasing all outcomes

                        
                            import pytest

                            def test_pass():
                                assert 1 + 1 == 2
                            
                            def test_fail():
                                assert 1 + 1 == 3
                            
                            @pytest.mark.skip(reason="Test not applicable in this environment")
                            def test_skip():
                                assert True

The test_pass function contains an assertion that is true, so the test will pass.

The test_fail function contains an assertion that is false, so the test will fail.

The test_skip function is marked with @pytest.mark.skip and will be skipped with the specified reason.

Parameterized Testing:

In Pytest, parameterized testing uses the @pytest.mark.parametrize decorator, which allows developers to run the same test with different sets of input parameters. This helps improve test coverage and the effectiveness of test suites.

                        
                            import pytest
                            from tic_tac_toe_ai import win_check
                            
                            @pytest.mark.parametrize("board, expected", [
                                (["X", "O", "X", "O", "X", "O", "O", "X", "X"], True),
                                (["O", "X", "O", "X", "O", "X", "X", "O", "O"], True),
                                (["X", "X", "O", "O", "O", "X", "X", "O", "X"], False),
                            ])
                            def test_win_check(board, expected):
                                result = win_check(board)
                                assert result == expected

In this case of parametrized testing with Pytest, the test_win_check function is systematically tested with different input scenarios (represented by the parameterized game boards), ensuring that the win_check function correctly evaluates whether a player has won the tic-tac-toe game. This approach improves test coverage and helps validate the functionality of the win_check function across various game configurations.
